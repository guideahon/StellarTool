[size=5]Stellar Tool — mod merger for Stellar Blade[/size]

Two mods edit the same table, the game loads only one, and you lose half of what you installed. Stellar Tool reads the DataTables inside your mods, shows every change, resolves conflicts, and builds one merged pak that keeps them all.

[b]Two ways to use it[/b]
[list]
[*][b]Simple:[/b] drop your mods, click [i]Merge everything[/i]. Conflicts auto-resolve by order, it installs to your ~mods, and offers to disable the originals.
[*][b]Advanced:[/b] every change as a checkbox, side-by-side conflict resolution, manual value editing, savable projects.
[/list]

[b]Every change is one readable line, vs vanilla:[/b]
[code]CharacterTable · ATL_M_Maelstrom_01 · MaxHP: 100000 → 300000 (+200%)
SkillTable · M_DroidTurret_Laser · AttackDamageRate: 1.5 → 4.5 (+200%)[/code]

[b]Reads Zen/IoStore mods[/b] (the usual Nexus .ucas/.utoc) via CUE4Parse — point it at your game once. Legacy .pak, .zip and loose .uasset folders too. On Zen mods, numeric changes (HP, damage, shields…) merge and verify; non-numeric ones are shown and reported, not silently dropped.

[b]Also[/b]: native Zen output, round-trip verified before packing; installable .zip + merge_report.txt; duplicate changes auto-collapsed; one-click vanilla baseline; headless CLI; 10 languages. Your source mods are never modified.

[b]Install[/b]
[list=1]
[*]Extract anywhere and run StellarTool.exe (self-contained, no installer).
[*]In Settings, confirm your Stellar Blade folder (auto-detected).
[*]Drop mods, review, Merge everything → installs to [code]SB\Content\Paks\~mods[/code].
[*][b]Disable the source mods you merged[/b] (one-click button) so they don't override the merge.
[/list]

Windows 10/11. No game file is ever touched. Open source (MIT): [url=https://github.com/guideahon/StellarTool]github.com/guideahon/StellarTool[/url] — you can compile it yourself.

[b]Bundled[/b] (unmodified open-source CLIs it drives): repak + retoc (trumank), UAssetGUI (atenfyr), cue4parse (CUE4Parse / FModel team). Thanks also to the SB modding community for the .usmap groundwork, and to Raxdiam whose "we're all editing the same file and clashing" note inspired this.
